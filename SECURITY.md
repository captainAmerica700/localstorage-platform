# Security Policy

## Supported versions

The latest release line is currently supported for security updates.

## Reporting a vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead, report them privately through GitHub's private vulnerability reporting feature for this repository if it is available. If that mechanism is not available, contact the maintainers directly through the project's GitHub profile and provide as much detail as possible.

Please include:

- a clear description of the issue
- steps to reproduce it
- the affected version
- any relevant proof-of-concept code or logs

We ask that you give the maintainers a reasonable amount of time to investigate and address the issue before publicly disclosing it.
